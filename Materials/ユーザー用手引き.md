## 1. はじめに

### 1.1 本システムについて

PRM Tool（Partner Relationship Management Tool）は、パートナー企業との関係管理を効率化するための統合Webアプリケーションです。

### 1.2 主な機能

- 👥 **アカウント管理**: ユーザーの作成・編集・削除、ロール管理
- 🏢 **パートナー管理**: パートナー企業と担当者の情報管理
- 📊 **案件管理**: 案件の進捗管理と担当者アサイン、スプレッドシート機能
- 💰 **手数料ルール管理**: 手数料ルールの作成・確定
- 📄 **請求書管理**: 請求書の作成・発行・支払管理
- 📋 **請求書テンプレート管理**: カスタマイズ可能な請求書テンプレート
- 📈 **ダッシュボード**: 統計情報の可視化

### 1.3 推奨環境

- **ブラウザ**: Google Chrome（最新版）、Microsoft Edge（最新版）、Safari（最新版）
- **画面解像度**: 1366x768 以上
- **インターネット接続**: 必須

## 2. システムの概要

### 2.1 画面構成

システムは以下の画面で構成されています：

1. **ログイン画面**: システムへのログイン
2. **ダッシュボード**: 統計情報の表示
3. **パートナー管理画面**: パートナー企業と担当者の管理
4. **案件管理画面**: 案件の管理とスプレッドシート
5. **手数料ルール管理画面**: 手数料ルールの管理
6. **請求書管理画面**: 請求書の管理
7. **請求書テンプレート管理画面**: 請求書テンプレートの管理
8. **アカウント管理画面**: ユーザーアカウントの管理

### 2.2 ナビゲーションバー

画面上部のナビゲーションバーから各機能にアクセスできます。

```
┌─────────────────────────────────────────────────────────────────┐
│ PRM Tool │ ダッシュボード │ パートナー │ 案件 │ 手数料管理 │
│ 請求書 │ アカウント管理 │ [ユーザー名] ログアウト │
└─────────────────────────────────────────────────────────────────┘
```

**注意**: 表示されるメニューはユーザーのロールによって異なります。

## 3. ログインとログアウト

### 3.1 ログイン方法

1. ブラウザでシステムのURLにアクセスします
2. ログイン画面が表示されます
3. **ログインID**と**パスワード**を入力します
4. **ログイン**ボタンをクリックします

```
┌──────────────────────────┐
│   PRM Tool ログイン      │
├──────────────────────────┤
│ ログインID: [_________] │
│ パスワード: [_________] │
│                          │
│      [ログイン]          │
└──────────────────────────┘
```

**初回ログイン時の注意**:
- システム管理者から提供されたログインIDとパスワードを使用してください
- 初回ログイン後、必ずパスワードを変更してください

### 3.2 ログアウト方法

1. 画面右上の**ログアウト**ボタンをクリックします
2. ログイン画面に戻ります

**セキュリティのために**:
- 作業を終えたら必ずログアウトしてください
- 長時間席を離れる場合もログアウトしてください

## 5. アカウント管理

**対象ロール**: SYSTEM、ADMIN

### 5.1 ユーザー一覧の表示

1. ナビゲーションバーから**アカウント管理**をクリックします
2. 登録されているユーザーの一覧が表示されます

表示される情報：
- 名前
- ログインID
- メールアドレス
- ロール
- 作成日時

### 5.2 新規ユーザーの作成

1. **+ 新規ユーザー**ボタンをクリックします
2. ユーザー情報を入力します：

**必須項目（*印）**:
- 名前 *
- ログインID * （英数字のみ、一意である必要があります）
- パスワード * （8文字以上）
- パスワード確認 *
- メールアドレス *
- ロール *

3. **作成**ボタンをクリックします

**パスワード要件**:
- 8文字以上
- 英字と数字を含むことを推奨

### 5.3 ユーザー情報の編集

1. ユーザーの行の**編集**ボタンをクリックします
2. 情報を修正します
3. **更新**ボタンをクリックします

**注意事項**:
- ログインIDは変更できません
- パスワードを変更する場合は、新しいパスワードと確認を入力してください

### 5.4 ユーザーの削除

1. ユーザーの行の**削除**ボタンをクリックします
2. 確認ダイアログが表示されます
3. **OK**をクリックすると削除されます

**重要な注意**:
- 削除されたユーザーは復元できません
- そのユーザーが作成した案件やデータは残りますが、所有者情報は失われます

## 6. パートナー管理

**対象ロール**: ADMIN、ACCOUNTING、REP（削除はADMINのみ）

### 6.1 パートナー一覧の表示

1. ナビゲーションバーから**パートナー**をクリックします
2. 登録されているパートナーの一覧が表示されます

表示される情報：
- パートナー名
- 代表者名
- 電話番号
- 担当者数
- 作成日

### 6.2 パートナーの検索

検索バーにキーワードを入力すると、以下の項目で検索できます：
- パートナー名
- 代表者名
- 住所
- 電話番号
- メールアドレス

### 6.3 新規パートナーの作成

1. **+ 新規パートナー**ボタンをクリックします
2. パートナー情報を入力します：

**基本情報（必須項目 *印）**:
- パートナー名 *
- 代表者名 *

**連絡先情報（任意）**:
- 郵便番号（ハイフンなし、7桁）
  - 入力後、自動で住所が検索されます
- 住所
- 電話番号
- メールアドレス

**郵便番号自動検索機能**:
- 郵便番号を入力すると、自動的に住所が検索されます
- 検索結果が複数ある場合、リストから選択できます
- 住所は手動で編集することもできます

3. **作成**ボタンをクリックします

### 6.4 パートナー情報の編集

1. パートナーの行の**編集**ボタンをクリックします
2. 情報を修正します
3. **更新**ボタンをクリックします

### 6.5 パートナーの削除

**対象ロール**: ADMIN のみ

1. パートナーの行の**削除**ボタンをクリックします
2. 確認ダイアログが表示されます
3. **OK**をクリックすると削除されます

**注意**:
- パートナーに関連する案件が存在する場合、削除できません
- 先に関連する案件を削除する必要があります

### 6.6 パートナー担当者の管理

パートナーには複数の担当者を登録できます。

#### 6.6.1 担当者の追加

1. パートナーの**詳細**をクリックします
2. 「担当者」セクションで**+ 担当者を追加**ボタンをクリックします
3. 担当者情報を入力します：

**担当者情報（必須項目 *印）**:
- 姓 *
- 名 *
- 役職
- メールアドレス
- 電話番号

4. **追加**ボタンをクリックします

#### 6.6.2 担当者の編集

1. 担当者の行の**編集**ボタンをクリックします
2. 情報を修正します
3. **更新**ボタンをクリックします

#### 6.6.3 担当者の削除

1. 担当者の行の**削除**ボタンをクリックします
2. 確認ダイアログが表示されます
3. **OK**をクリックすると削除されます

**注意**:
- 担当者情報は請求書などで使用される場合があります
- 削除前に、使用されていないことを確認してください

## 7. 案件管理

**対象ロール**: ADMIN、ACCOUNTING、REP（削除はADMINとACCOUNTINGのみ）

### 7.1 案件一覧の表示

1. ナビゲーションバーから**案件**をクリックします
2. 登録されている案件の一覧が表示されます

表示される情報：
- 案件名
- パートナー名
- ステータス
- オーナー（作成者）
- 担当者数
- 作成日

### 7.2 案件の検索とフィルター

#### 検索機能

検索バーにキーワードを入力すると、以下の項目で検索できます：
- 案件名
- パートナー名

#### フィルター機能

ステータスとパートナーでフィルタリングできます：
- ステータス: 全て / 新規 / 進行中 / 完了
- パートナー: 全てのパートナー / 特定のパートナー

### 7.3 案件の表示権限

**権限の種類**:

| ステータス | 表示ルール |
|-----------|-----------|
| **新規（NEW）** | すべてのユーザーが閲覧可能 |
| **進行中（IN_PROGRESS）** | 担当者のみ閲覧可能 |
| **完了（DONE）** | 担当者のみ閲覧可能 |

**注意**:
- ADMIN と ACCOUNTING は全案件を閲覧可能
- REP は新規案件と自分が担当している案件のみ閲覧可能

### 7.4 新規案件の作成

1. **+ 新規案件**ボタンをクリックします
2. 案件情報を入力します：

**必須項目（*印）**:
- 案件名 *
- パートナー *（ドロップダウンから選択）
- ステータス *

3. **担当者をアサイン**します：
   - 検索バーでユーザーを検索できます
   - チェックボックスで複数の担当者を選択できます

4. **作成**ボタンをクリックします

**注意事項**:
- 案件を作成したユーザーが自動的に「オーナー」として設定されます
- オーナーは担当者一覧に自動的に含まれます

### 7.5 案件情報の編集

1. 案件の行の**編集**ボタンをクリックします
2. 情報を修正します
3. **更新**ボタンをクリックします

### 7.6 案件の削除

**対象ロール**: ADMIN、ACCOUNTING のみ

1. 案件の行の**削除**ボタンをクリックします
2. 確認ダイアログが表示されます
3. **OK**をクリックすると削除されます

**注意**:
- 案件に関連する手数料ルールや請求書が存在する場合、削除できません
- 先に関連するデータを削除する必要があります

### 7.7 案件詳細とスプレッドシート機能

案件詳細画面では、案件に関する詳細情報をスプレッドシート形式で管理できます。

#### 7.7.1 詳細画面へのアクセス

1. 案件一覧で**詳細**ボタンをクリックします
2. 案件詳細画面が表示されます

#### 7.7.2 スプレッドシート機能

**機能**:
- Excel風のスプレッドシート
- データの入力・編集
- 行の追加・削除
- 列の追加・削除
- データの自動保存

**使い方**:
1. セルをクリックして編集します
2. 行を追加する場合は、**行を追加**ボタンをクリックします
3. 列を追加する場合は、**列を追加**ボタンをクリックします
4. データは自動的に保存されます

**注意**:
- スプレッドシートのデータはJSON形式で保存されます
- 大量のデータを入力すると、表示が遅くなる場合があります

## 8. 手数料ルール管理

**対象ロール**: 
- 作成・編集・確定: ADMIN、ACCOUNTING
- 削除: ADMIN のみ
- 閲覧: 全ロール

### 8.1 手数料ルールとは

手数料ルールは、請求書で使用する手数料の計算方法を定義するものです。

### 8.2 手数料ルール一覧の表示

1. ナビゲーションバーから**手数料管理**をクリックします
2. 登録されている手数料ルールの一覧が表示されます

表示される情報：
- ルール名
- 案件名
- パートナー名
- 手数料タイプ
- ステータス
- 作成日

### 8.3 手数料ルールのステータス

| ステータス | 説明 | 請求書での使用 |
|-----------|------|--------------|
| **下書き（DRAFT）** | 作成中のルール | 使用不可 |
| **確定（CONFIRMED）** | 確定済のルール | 使用可能 |

**重要**:
- 請求書で使用できるのは「確定」状態のルールのみです
- ルールを確定すると、編集できなくなります

### 8.4 新規手数料ルールの作成

1. **+ 新規ルール**ボタンをクリックします
2. ルール情報を入力します：

**必須項目（*印）**:
- ルール名 *
- 案件 *（ドロップダウンから選択）
- 手数料タイプ *
- 適用条件（タイプによって異なります）

**手数料タイプの種類**:

| タイプ | 説明 | 必要な入力項目 |
|--------|------|--------------|
| **固定金額（FIXED）** | 数量に関係なく固定の金額 | 固定金額（円） |
| **割合（RATE）** | 商品金額に対する割合 | 手数料率（%） |

3. **作成**ボタンをクリックします

### 8.5 手数料ルールの編集

**注意**: 確定済のルールは編集できません

1. 下書き状態のルールの**編集**ボタンをクリックします
2. 情報を修正します
3. **更新**ボタンをクリックします

### 8.6 手数料ルールの確定

**対象ロール**: ADMIN、ACCOUNTING

1. 下書き状態のルールの**確定**ボタンをクリックします
2. 確認ダイアログが表示されます
3. **OK**をクリックすると確定されます

**重要な注意**:
- 確定後は編集できなくなります
- 確定済のルールのみ請求書で使用できます
- 確定前に内容をよく確認してください

### 8.7 手数料ルールの削除

**対象ロール**: ADMIN のみ

1. ルールの**削除**ボタンをクリックします
2. 確認ダイアログが表示されます
3. **OK**をクリックすると削除されます

**注意**:
- 請求書で使用されているルールは削除できません
- 確定済のルールを削除する場合は特に注意してください

### 8.8 手数料の計算例

#### 例1: 固定金額タイプ

**設定**:
- ルール名: 基本手数料
- タイプ: 固定金額
- 固定金額: 10,000円

**計算**:
- 数量1個: 手数料 10,000円
- 数量5個: 手数料 10,000円
- 数量10個: 手数料 10,000円

#### 例2: 割合タイプ

**設定**:
- ルール名: 販売手数料
- タイプ: 割合
- 手数料率: 10%

**計算**:
- 商品金額 100,000円: 手数料 10,000円
- 商品金額 500,000円: 手数料 50,000円
- 商品金額 1,000,000円: 手数料 100,000円

## 9. 請求書管理

**対象ロール**: 
- 作成・編集・発行: ADMIN、ACCOUNTING
- 削除: ADMIN のみ
- 閲覧: 全ロール

### 9.1 請求書一覧の表示

1. ナビゲーションバーから**請求書**をクリックします
2. 登録されている請求書の一覧が表示されます

表示される情報：
- 請求書番号
- パートナー名
- 発行日
- 合計金額
- ステータス
- 作成日

### 9.2 請求書のステータス

| ステータス | 説明 | 次のアクション |
|-----------|------|--------------|
| **下書き（DRAFT）** | 作成中の請求書 | 編集・発行 |
| **発行済（ISSUED）** | 発行された請求書 | 支払済に変更 |
| **支払済（PAID）** | 支払いが完了した請求書 | - |
| **キャンセル（CANCELLED）** | キャンセルされた請求書 | - |

**ステータスの遷移**:
```
下書き → 発行済 → 支払済
   ↓
キャンセル
```

### 9.3 新規請求書の作成

1. **+ 新規請求書**ボタンをクリックします
2. 請求書情報を入力します：

#### ステップ1: 基本情報の入力

**必須項目（*印）**:
- パートナー *（ドロップダウンから選択）
- 発行日 *
- 請求書番号 *
- テンプレート（任意）

**パートナー整合性チェック**:
- パートナーを選択すると、そのパートナーに紐づく手数料ルールのみが選択可能になります
- パートナーを変更すると、手数料ルールの選択は自動的にクリアされます

#### ステップ2: 明細の追加

**明細項目（*印は必須）**:
- 説明 *（商品名やサービス名）
- 数量 *
- 単価 *
- 手数料ルール（任意）

**計算の仕組み**:
1. 商品金額 = 単価 × 数量
2. 手数料額 = 選択した手数料ルールに基づいて自動計算
3. 明細合計 = 商品金額 + 手数料額

**明細の追加方法**:
1. **+ 明細を追加**ボタンをクリックします
2. 明細情報を入力します
3. 手数料ルールを選択します（任意）
4. 複数の明細を追加できます

#### ステップ3: 合計の確認

システムが自動的に以下を計算します：
- 小計（商品金額の合計）
- 手数料小計（手数料の合計）
- 消費税（10%）
- 合計金額

3. **作成**ボタンをクリックします

### 9.4 請求書の編集

**注意**: 下書き状態の請求書のみ編集可能

1. 下書き状態の請求書の**編集**ボタンをクリックします
2. 情報を修正します
3. **更新**ボタンをクリックします

**編集できない項目**:
- 発行済・支払済・キャンセル状態の請求書は編集できません

### 9.5 請求書の発行

**対象ロール**: ADMIN、ACCOUNTING

1. 下書き状態の請求書の**発行**ボタンをクリックします
2. 確認ダイアログが表示されます
3. **OK**をクリックすると発行されます

**発行時の処理**:
- ステータスが「発行済」に変更されます
- 編集ができなくなります
- PDF出力が可能になります

### 9.6 請求書を「支払済」に変更

**対象ロール**: ADMIN、ACCOUNTING

1. 発行済の請求書の**支払済に変更**ボタンをクリックします
2. 確認ダイアログが表示されます
3. **OK**をクリックすると支払済になります

**注意**:
- 発行済の請求書のみ支払済に変更できます
- 支払済に変更すると、元に戻せません

### 9.7 請求書の削除

**対象ロール**: ADMIN のみ

1. 請求書の**削除**ボタンをクリックします
2. 確認ダイアログが表示されます
3. **OK**をクリックすると削除されます

**注意**:
- 発行済の請求書を削除する場合は特に注意してください
- 削除された請求書は復元できません

### 9.8 請求書のPDF出力

1. 請求書の**PDFプレビュー**ボタンをクリックします
2. 新しいタブでPDFが表示されます
3. ブラウザの印刷機能またはダウンロード機能を使用できます

## 12. よくある質問（FAQ）

### Q1. パスワードを忘れてしまいました

**A**: システム管理者に連絡して、パスワードをリセットしてもらってください。

---

### Q2. 手数料ルールを確定してしまいましたが、内容を変更したいです

**A**: 確定済のルールは編集できません。以下の対応方法があります：

1. **新しいルールを作成する**（推奨）
   - 既存のルールは残しておき、新しいルールを作成してください
   - 今後の請求書では新しいルールを使用してください

2. **ルールを削除して作り直す**（請求書で未使用の場合のみ）
   - そのルールが請求書で使用されていない場合のみ、削除して作り直せます
   - 使用されている場合は削除できません

---

### Q3. 請求書を発行してしまいましたが、内容に誤りがありました

**A**: 以下の手順で対応してください：

1. **発行済の請求書を削除する**（ADMIN権限が必要）
2. **新しい請求書を作成し直す**
3. **正しい内容で発行する**

**注意**: 削除された請求書は復元できないため、削除前に内容を確認してください。

---

### Q4. 案件を削除しようとするとエラーが表示されます

**A**: 案件に関連する以下のデータが存在する場合、削除できません：

- 手数料ルール
- 請求書

**対処法**:
1. まず、関連する手数料ルールと請求書をすべて削除します
2. その後、案件を削除できます

---

### Q5. パートナーを変更したら手数料ルールが選択できなくなりました

**A**: これは正常な動作です。パートナー整合性チェック機能により、選択したパートナーに紐づく手数料ルールのみが表示されます。

**対処法**:
1. 正しいパートナーを選択してください
2. そのパートナーに紐づく手数料ルールを選択してください

---

### Q6. REPロールですが、案件が表示されません

**A**: REPロールの場合、以下の案件のみ表示されます：

- ステータスが「新規（NEW）」の案件
- 自分が担当者として割り当てられている案件

**確認事項**:
- 案件のステータスを確認してください
- 自分が担当者に追加されているか確認してください

---

### Q7. スプレッドシートのデータが消えてしまいました

**A**: スプレッドシートのデータは自動保存されますが、以下の場合は保存されません：

- ブラウザがクラッシュした場合
- インターネット接続が切断された場合

**対処法**:
- 定期的に**保存**ボタンをクリックして手動保存してください
- 重要なデータは別途バックアップを取ってください

---

## 13. トラブルシューティング

### 問題1: ログインできない

**症状**: ログインIDとパスワードを入力しても「認証に失敗しました」と表示される

**原因**:
- ログインIDまたはパスワードが間違っている
- アカウントが無効化されている
- CapsLockがオンになっている

**対処法**:
1. ログインIDとパスワードを確認してください
2. CapsLockがオフになっているか確認してください
3. それでも解決しない場合は、システム管理者に連絡してください

---

### 問題2: 画面が真っ白になる

**症状**: ログイン後、画面が真っ白になって何も表示されない

**原因**:
- JavaScriptエラー
- ブラウザのキャッシュ
- ネットワークエラー

**対処法**:
1. ブラウザを再読み込みしてください（Ctrl + F5 または Cmd + Shift + R）
2. ブラウザのキャッシュをクリアしてください
3. 別のブラウザで試してください
4. それでも解決しない場合は、システム管理者に連絡してください

---

### 問題3: データが保存されない

**症状**: データを入力して保存ボタンをクリックしても保存されない

**原因**:
- 必須項目が未入力
- バリデーションエラー
- ネットワークエラー
- サーバーエラー

**対処法**:
1. すべての必須項目（*印）が入力されているか確認してください
2. エラーメッセージが表示されていないか確認してください
3. インターネット接続を確認してください
4. ブラウザのコンソールでエラーがないか確認してください
5. それでも解決しない場合は、システム管理者に連絡してください

---

### 問題4: PDFが表示されない

**症状**: PDFプレビューボタンをクリックしても何も表示されない

**原因**:
- ブラウザのポップアップブロック
- PDFビューアーの設定
- サーバーエラー

**対処法**:
1. ブラウザのポップアップブロックを無効にしてください
2. ブラウザのPDFビューアー設定を確認してください
3. 別のブラウザで試してください
4. それでも解決しない場合は、システム管理者に連絡してください

---

### 問題5: 手数料ルールが表示されない

**症状**: 請求書作成時、手数料ルールのドロップダウンが空になっている

**原因と対処法**:

**1. パートナーが未選択**
   - パートナーを先に選択してください
   - パートナー選択前は手数料ルールを選択できません

**2. 該当する手数料ルールがない**
   - 選択したパートナーに紐づく確定済の手数料ルールが存在しません
   - 以下を確認してください：
     - そのパートナー向けの案件が存在するか
     - その案件に紐づく手数料ルールが存在するか
     - 手数料ルールが「確定済」ステータスになっているか

**3. 対処方法**:
   - 管理者または会計担当ロールのユーザーに連絡してください
   - 必要な案件と手数料ルールの作成・確定を依頼してください

---

### 問題6: 請求書作成時にエラーが表示される

**症状**: 「手数料ルール「XX」は別のパートナー向けです」というエラーが表示される

**原因**: パートナーと手数料ルールの不整合

**対処法**:
1. ブラウザを再読み込みしてください（Ctrl + F5 または Cmd + Shift + R）
2. パートナーを再選択してください
3. パートナーに紐づく手数料ルールを選択してください
4. それでも解決しない場合は、システム管理者に連絡してください

**注意**:
- このエラーはデータの整合性を保つための保護機能です
- 正しいパートナーと手数料ルールの組み合わせを選択してください

---

### 問題7: 案件のスプレッドシートが動作しない

**症状**: スプレッドシートにデータを入力できない、または表示されない

**原因**:
- JavaScriptエラー
- ブラウザの互換性
- データの破損

**対処法**:
1. ブラウザを再読み込みしてください
2. Google Chrome（最新版）を使用してください
3. ブラウザのコンソールでエラーがないか確認してください
4. それでも解決しない場合は、システム管理者に連絡してください
